# C Build System Flow

This document explains the complete build pipeline for the ANGIO2D C solver.

## Overview: From Source to Executable

```
Source Code (src/*.c)
    ↓
    ├─ build_config.mk (compiler flags)
    ├─ Makefile (build rules)
    └─ include/*.h (headers)
    ↓
Compilation & Linking
    ↓
Executable (build/angio2d)
    ↓
Execution
    ↓
Output
    ├─ diagnostics_c.csv
    ├─ solution_c_*.csv
    └─ [figures generated by Python]
```

---

## File Roles & Structure

### 1. **build_config.mk** — Centralized Compiler Configuration

**Purpose:** Define all compiler flags in one place to avoid repetition and environment issues.

**Location:** `angio2d_c/build_config.mk`

**Contents:**

```makefile
# Compiler
CC = cc

# Language standard
CSTD = -std=c99

# Warning flags
WARN_FLAGS = -Wall -Wextra -Wpedantic

# Optimization
OPT_FLAGS = -O2

# Preprocessor: Include paths
CPPFLAGS = 
CPPFLAGS += -I include

# Linker: Library paths
LDFLAGS = 
LDFLAGS += -L/opt/homebrew/opt/openssl@3/lib

# Linker: Libraries to link
LDLIBS = -lm
```

**Why separate flags?**

| Flag Type | What It Does | Example |
|-----------|--------------|---------|
| `CSTD` | Language version | `-std=c99` |
| `WARN_FLAGS` | Compiler warnings | `-Wall -Wextra` |
| `OPT_FLAGS` | Optimization level | `-O2` (speed) or `-O0` (debugging) |
| `CPPFLAGS` | Preprocessor includes | `-I include` (find `#include <file.h>`) |
| `LDFLAGS` | Linker search paths | `-L /path/to/lib` |
| `LDLIBS` | Libraries to link | `-lm` (math library), `-lpthread`, etc. |

**Why use `+=` instead of `=`?**

```makefile
# ❌ WRONG: Overwrites any system CPPFLAGS
CPPFLAGS = -I include

# ✅ CORRECT: Appends to existing CPPFLAGS
CPPFLAGS += -I include
```

On macOS, Homebrew sets environment variables like:
```bash
export CPPFLAGS="-I/opt/homebrew/opt/openssl@3/include"
```

Using `+=` ensures we **keep Homebrew's flags AND add ours**, not replace them.

### 2. **Makefile** — Build Rules & Orchestration

**Purpose:** Define compilation rules, dependencies, and build targets.

**Location:** `angio2d_c/Makefile`

**Key Sections:**

#### a) Include Configuration
```makefile
include build_config.mk
```
Pulls in all compiler flags defined above.

#### b) Directory & File Setup
```makefile
BUILD_DIR = build
SRC_DIR = src
INCLUDE_DIR = include

SOURCES = $(wildcard $(SRC_DIR)/*.c)    # Find all .c files
OBJECTS = $(SOURCES:$(SRC_DIR)/%.c=$(BUILD_DIR)/%.o)  # Map to .o files
EXECUTABLE = $(BUILD_DIR)/angio2d
```

**Example:**
- `src/main.c` → `build/main.o`
- `src/grid.c` → `build/grid.o`
- `src/adi.c` → `build/adi.o`
- etc.

#### c) Compilation Rule (Source → Object)
```makefile
$(BUILD_DIR)/%.o: $(SRC_DIR)/%.c
	@mkdir -p $(BUILD_DIR)
	$(CC) $(CPPFLAGS) $(CFLAGS) -c $< -o $@
```

**What it does:**
- `$(CC)` = compiler (cc)
- `$(CPPFLAGS)` = preprocessor flags (`-I include`)
- `$(CFLAGS)` = compilation flags (CSTD + WARN_FLAGS + OPT_FLAGS)
- `-c` = compile only (no linking)
- `$<` = input file (e.g., `src/main.c`)
- `$@` = output file (e.g., `build/main.o`)

**Real command executed:**
```bash
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/main.c -o build/main.o
```

#### d) Linking Rule (Objects → Executable)
```makefile
$(EXECUTABLE): $(OBJECTS)
	$(CC) $(LDFLAGS) -o $@ $^ $(LDLIBS)
```

**What it does:**
- Takes all `.o` files
- Links them together with `-lm` (math library)
- Produces executable `build/angio2d`

**Real command executed:**
```bash
cc -L/opt/homebrew/opt/openssl@3/lib -o build/angio2d \
  build/main.o build/grid.o build/adi.o ... -lm
```

#### e) Special Targets
```makefile
all: $(EXECUTABLE)        # Default: build executable
clean:                     # Remove build directory
	rm -rf $(BUILD_DIR)
.PHONY: all clean          # Mark as non-file targets
```

**Usage:**
```bash
make           # Build (calls 'all' by default)
make clean     # Clean build directory
make clean && make  # Rebuild from scratch
```

### 3. **Source Code Organization** — `src/` and `include/`

**Location:** `angio2d_c/src/` and `angio2d_c/include/`

**Module Structure:**

| File | Role |
|------|------|
| `src/params.c` + `include/params.h` | Grid dimensions, timesteps, parameters |
| `src/grid.c` + `include/grid.h` | Spatial grid creation and management |
| `src/operators.c` + `include/operators.h` | Laplacian, gradient, boundary conditions |
| `src/adi.c` + `include/adi.h` | ADI time-stepping solver |
| `src/reaction.c` + `include/reaction.h` | Reaction kinetics (source terms) |
| `src/taf.c` + `include/taf.h` | TAF field computation |
| `src/diagnostics.c` + `include/diagnostics.h` | Mass/energy tracking |
| `src/output.c` + `include/output.h` | CSV file writing |
| `src/main.c` | Solver entry point (orchestrates everything) |

**Compilation Order (Makefile handles automatically):**
1. Compile all `src/*.c` to `build/*.o`
2. Link all `build/*.o` → `build/angio2d`

**Header Dependencies:**
```
main.c
├─ #include "params.h"
├─ #include "grid.h"
├─ #include "operators.h"
├─ #include "adi.h"
├─ #include "reaction.h"
├─ #include "taf.h"
├─ #include "diagnostics.h"
└─ #include "output.h"
```

If you modify a header, Makefile **should rebuild affected .o files** (if using proper dependencies).

## Complete Build Flow (Step-by-Step)

### Example: Running `make`

```bash
$ cd angio2d_c
$ make clean
```
→ Removes `build/` directory completely

```bash
$ make
```

**Step 1: Check if `build/` exists**
```bash
mkdir -p build
```

**Step 2: Compile each .c file**
```bash
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/params.c -o build/params.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/grid.c -o build/grid.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/taf.c -o build/taf.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/operators.c -o build/operators.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/reaction.c -o build/reaction.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/adi.c -o build/adi.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/diagnostics.c -o build/diagnostics.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/output.c -o build/output.o
cc -I include -std=c99 -Wall -Wextra -Wpedantic -O2 -c src/main.c -o build/main.o
```

**Result:** 9 `.o` files in `build/`

**Step 3: Link everything into executable**
```bash
cc -L/opt/homebrew/opt/openssl@3/lib -o build/angio2d \
  build/params.o build/grid.o build/taf.o build/operators.o \
  build/reaction.o build/adi.o build/diagnostics.o build/output.o \
  build/main.o -lm
```

**Result:** `build/angio2d` executable

### Speed: Incremental Compilation

If you change **only** `src/adi.c`:
```bash
$ make
```

Makefile detects only `adi.c` changed → recompiles only `build/adi.o` → relinks executable.

Other `.o` files are **not recompiled** (faster!).

## Environment Variables & Customization

### Changing Compiler Flags

**To optimize for speed (instead of balance):**
```makefile
# In build_config.mk
OPT_FLAGS = -O3   # More aggressive optimization
```

**To add debug symbols (for GDB):**
```makefile
OPT_FLAGS = -O0 -g   # No optimization, debug info
```

**To add more warnings:**
```makefile
WARN_FLAGS = -Wall -Wextra -Wpedantic -Werror   # Treat warnings as errors
```

### Adding New Libraries

If you need to link against `libpng`:
```makefile
# In build_config.mk
LDFLAGS += -L/usr/local/lib    # Path to libpng
LDLIBS += -lpng                 # Library name
```

### Switching Compilers

```makefile
# Default: clang (macOS)
CC = cc

# Switch to GCC if installed:
CC = gcc

# Or explicit:
CC = /usr/bin/clang
```

## Makefile Best Practices

### ✅ DO

- Use `+=` for flags to preserve environment variables
- Keep build artifacts in `build/` directory (not in source)
- Use `$(wildcard)` to auto-detect source files
- Mark targets like `clean` as `.PHONY` (not file targets)
- Separate compilation and linking phases

### ❌ DON'T

- Hardcode compiler flags in Makefile (use `build_config.mk`)
- Mix `=` and `+=` for the same variable (inconsistent behavior)
- Put build artifacts in `src/` (clutters source)
- Force rebuild with `rm -f *.o` (inefficient)

## Integration with Python Pipeline

The full workflow is:

```
1. Python: run_pipeline.py
   ↓
2. Calls: make clean
   → Removes build/
   ↓
3. Calls: make
   → Compiles C code
   → Produces build/angio2d
   ↓
4. Calls: ./build/angio2d
   → Executes solver
   → Produces diagnostics_c.csv, solution_c_*.csv
   ↓
5. Python: plot_results.py
   → Reads CSV files
   → Generates 4 figures in output-c/
   ↓
6. Python: compare_with_matlab.py
   → Compares C figures vs MATLAB reference
   → Prints MAE/RMSE metrics
```

Each step depends on the previous one's output.

## Troubleshooting

### Problem: "fatal error: params.h: No such file or directory"
**Cause:** `-I include` missing from CPPFLAGS
**Fix:** Check `build_config.mk` has `CPPFLAGS += -I include`

### Problem: "undefined reference to `sqrt`"
**Cause:** Math library `-lm` not linked
**Fix:** Check `build_config.mk` has `LDLIBS = -lm`

### Problem: Compilation slow, rebuilds everything
**Cause:** Headers changed; all `.o` files invalidated
**Fix:** Make sure you're only modifying source (not headers if possible)

### Problem: Old executable still runs even after code change
**Cause:** Makefile not detecting change, or old binary in PATH
**Fix:** `make clean && make` to force full rebuild

## Summary

| Component | Why | Example |
|-----------|-----|---------|
| `build_config.mk` | Centralize flags; prevent environment collisions | `CPPFLAGS += -I include` |
| `Makefile` | Define build rules; enable incremental compilation | Compile `.c` → `.o` → link |
| `src/*.c` | Source code modules | Grid, ADI solver, I/O, etc. |
| `include/*.h` | Public interfaces | Function declarations, structs |
| `build/` | Build artifacts (not in git) | `.o` files, executable |

The system is **modular, efficient, and portable** across macOS, Linux, and other Unix-like systems.
